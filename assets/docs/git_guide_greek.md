# Οδηγός Χρήσης Git & GitHub (Στα Ελληνικά)

Αυτός ο οδηγός θα σας βοηθήσει να διαχειρίζεστε τις αλλαγές σας στο project άφοβα και με ασφάλεια.

---

## 1. Τι είναι το Branch (Κλαδί);
Σκεφτείτε το σαν ένα "παράλληλο σύμπαν" του κώδικα. Ό,τι κάνετε σε ένα branch δεν επηρεάζει το κυρίως project (που συνήθως λέγεται `main` ή `master`) μέχρι να το επιλέξετε εσείς.

## 2. Δημιουργία και Μετάβαση σε Νέο Branch
Για να ξεκινήσετε μια νέα δοκιμή ή να προσθέσετε κάτι καινούργιο:

```bash
git checkout -b to-onoma-tou-branch-sas
```
*   Το `-b` σημαίνει "δημούργησε το κλαδί".
*   Το `checkout` σημαίνει "μετάφερέ με σε αυτό".

**Παράδειγμα:**
`git checkout -b geomaras-testing`

---

## 3. Πώς να δείτε σε ποιο Branch βρίσκεστε
Τρέξτε την εντολή:
```bash
git branch
```
Το branch με το αστεράκι `*` (και συνήθως με πράσινο χρώμα) είναι αυτό στο οποίο δουλεύετε αυτή τη στιγμή.

---

## 4. Επιστροφή στον Αρχικό Κώδικα (Main)
Αν θέλετε να "βγείτε" από τις δοκιμές σας και να δείτε τον αρχικό κώδικα:
```bash
git checkout main
```
*Προσοχή: Πριν αλλάξετε branch, καλό είναι να έχετε αποθηκεύσει τις αλλαγές σας (δείτε παρακάτω).*

---

## 5. Αποθήκευση των Αλλαγών σας (Commit)
Για να "κλειδώσετε" τις αλλαγές που κάνατε στο δικό σας branch:

1. Ενημερώνετε το git για το ποια αρχεία αλλάξατε:
   ```bash
   git add .
   ```
2. Αποθηκεύετε με ένα μήνυμα:
   ```bash
   git commit -m "Περιγραφή της αλλαγής μου"
   ```

---

## 6. Συγχώνευση (Merge)
Όταν είστε σίγουροι ότι οι δοκιμές στο branch σας είναι πετυχημένες και θέλετε να τις μεταφέρετε στον `main` κορμό:

1. Πηγαίνετε πρώτα στον `main`:
   ```bash
   git checkout main
   ```
2. Φέρτε τις αλλαγές από το δικό σας branch:
   ```bash
   git merge to-onoma-tou-branch-sas
   ```

---

## 7. Πώς να δει ο Συνεργάτης τις αλλαγές σας
Αν θέλετε να ανεβάσετε το branch σας στο GitHub για να το δει ο συνεργάτης σας:
```bash
git push origin to-onoma-tou-branch-sas
```
Στη συνέχεια, στο site του GitHub θα δείτε ένα κουμπί **"Compare & pull request"**. Πατήστε το για να ζητήσετε επίσημα από τον συνεργάτη σας να ενσωματώσει τις αλλαγές σας.

---

## 8. Χρήσιμες Συμβουλές για Αρχάριους
*   **Πριν ξεκινήσετε δουλειά**, τρέχετε πάντα `git pull origin main` για να έχετε την πιο φρέσκια έκδοση του συνεργάτη σας.
*   **Μην φοβάστε να χαλάσετε κάτι στο δικό σας branch.** Μπορείτε πάντα να το διαγράψετε και να ξεκινήσετε από την αρχή!
*   **Προσοχή στις εντολές:** Αν το Git σας βγάλει σφάλμα, ελέγξτε αν γράψατε σωστά την εντολή (π.χ. `branch` και όχι `brance`).
*   Για να διαγράψετε ένα branch που δεν χρειάζεστε πια: `git branch -d onoma-branch`.

---

## 9. Μεταφορά σε άλλο Υπολογιστή (Clone & Checkout)
Αν θέλετε να συνεχίσετε τη δουλειά σας σε έναν άλλον υπολογιστή:

1. **Κατεβάστε το project (Clone):**
   ```bash
   git clone https://github.com/IoannisMaras/notes-gema.git
   ```
2. **Μπείτε στο φάκελο:**
   ```bash
   cd notes-gema
   ```
3. **Φέρτε το branch σας:**
   Αρχικά το Git κατεβάζει μόνο το `main`. Για να δείτε και το δικό σας (εφόσον το έχετε κάνει `push` στο GitHub):
   ```bash
   git fetch origin
   git checkout to-onoma-tou-branch-sas
   ```
4. **Εγκατάσταση Dependencies (Flutter):**
   ```bash
   flutter pub get
   ```
